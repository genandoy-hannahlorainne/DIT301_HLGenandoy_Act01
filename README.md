# Reflection

## 1. What challenges did I encounter installing Android Studio?
While installing Android Studio, I encountered several challenges. 
One of the main issues was setting up the correct JDK version because the installer showed errors when the target path already contained files. 
I also struggled with running the emulator since it sometimes froze on the Google boot screen or failed to install the app due to package errors. 
These challenges tested my patience, but they also taught me how to troubleshoot by changing installation paths, adjusting settings, and recreating the emulator until it worked properly.

## 2. Learnings about Android app structure
Through the installation and initial exploration of Android Studio, I learned that an Android app project is organized into several folders and files that serve different purposes. 
The `app/src/main/java` folder contains the Kotlin/Java code, while the `res` folder holds resources such as layouts, images, and strings. 
The **manifest file** defines the essential information about the app, such as activities and permissions. I also learned that **Gradle** plays an important role in managing dependencies and building the project. 
This structure makes it easier to separate code, resources, and configuration files, allowing developers to maintain and scale their apps effectively.
